# Installed Codex Skills

Short notes on installed skills that are useful for this project.

| Skill | Useful For This Project |
|---|---|
| `ast-grep` | Structural Python searches or codemods when text search is not enough. |
| `benchmark` | Measuring runtime regressions for simulations, examples, or Streamlit-heavy paths. |
| `benchmark-methodology` | Comparing project positioning or competitor/portfolio quality in a structured way. |
| `caveman` | Very compressed communication when context or token budget is tight. |
| `code-tour` | Creating `.tour` walkthrough artifacts with real file anchors for onboarding. |
| `codebase-onboarding` | Summarizing repo structure, entry points, conventions, and architecture for new chats. |
| `delivery-gate` | Final quality reminder to verify work, avoid skipped checks, and document outputs. |
| `developing-with-streamlit` | All Streamlit app work: debugging, layout, UI polish, state, performance, and CLI usage. |
| `e2e-testing` | Playwright-style smoke tests if the Streamlit UI needs browser-level verification. |
| `find-skills` | Discovering additional skills if future tasks need specialized workflows. |
| `grill-me` | Stress-testing a plan before a major architecture, roadmap, or validation push. |
| `matplotlib` | Low-level plot customization and figure debugging. |
| `numpy` | Array/vectorization practices for numerical code and tests. |
| `prompt-optimizer` | Turning rough future instructions into clear reusable prompts. |
| `python-patterns` | Python design, readability, module structure, typing, and refactoring guidance. |
| `python-testing` | Pytest tests, fixtures, parametrization, coverage strategy, and regression checks. |
| `ruff` | Python linting and formatting checks. |
| `scientific-visualization` | Plot polish, colorblind-friendly palettes, labels, figure export, and scientific readability. |
| `scipy-optimization-toolkit` | Optimization, integration, and signal-processing work involving SciPy. |
| `strategic-compact` | Creating clean handoffs and compacting context at phase boundaries. |
| `tdd-workflow` | Test-first behavior for bug fixes and new functionality. |
| `ty` | Python type checking if the project adds stricter typing. |
| `uv` | Python environment/package management if the project moves to uv. |
